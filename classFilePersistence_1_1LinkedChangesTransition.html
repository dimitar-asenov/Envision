Let keySet be the return value of this.findLinkedChanges(changeId,inBranchA).

When this method returns, keySet will be mapped to a LinkedChanges object containing change. All preexisting mappings persist. Details follow.

If keySet is already mapped to a LinkedChanges containing change, this is a no-op. If change is not already mapped to by some LinkedChanges and keySet is not already mapped to a LinkedChanges then keySet is mapped to a new LinkedChanges containing only change. If change is not already mapped to and keySet is already mapped to a LinkedChanges object, change is added to that object. If change is already mapped to by some LinkedChanges and keySet is not yet mapped to any LinkedChanges, keySet is mapped to the LinkedChanges containing change. If keySet is already mapped to a LinkedChanges current and a different LinkedChanges other already contains change, keySet and all LinkedChanges mapped to other are mapped to the union of other and current.
